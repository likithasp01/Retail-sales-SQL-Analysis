# Retail Sales Data Cleaning and Analysis (SQL)

## Problem Statement

Retail businesses generate large volumes of sales transactions every day. However, raw sales data often contains inconsistencies and requires cleaning before meaningful analysis can be performed. This project focuses on cleaning a retail sales dataset using SQL and performing business analysis to uncover insights into customer behavior, product performance, and sales trends.

---

## Dataset

The dataset contains retail transaction information, including:

* Transaction ID
* Customer ID
* Date
* Product Category
* Quantity
* Price per Unit
* Total Amount
* Gender
* Age

---

## Tools Used

* **MySQL**
* **SQL**
* **CSV Dataset**

---

## Process

###  Data Collection

* Imported the retail sales dataset into MySQL.

###  Data Cleaning

* Created a staging table to preserve the original dataset.
* Renamed columns for consistency and readability.
* Converted the Date column to the DATE data type.
* Checked for duplicate transaction records.
* Identified missing customer IDs.
* Verified invalid values such as negative quantities and prices.
* Prepared the dataset for business analysis.

###  Data Analysis

Performed SQL queries to analyze:

* Total sales revenue
* Total number of transactions
* Unique customers
* Average transaction value
* Revenue by product category
* Monthly sales performance
* Sales by gender
* Highest-value transactions
* Average quantity purchased
* Revenue by customer age group

---

## Key Findings

* Calculated the overall sales revenue generated by the business.
* Identified the number of unique customers and total transactions.
* Determined the highest revenue-generating product categories.
* Analyzed monthly revenue trends to identify sales patterns.
* Compared purchasing behavior across gender and age groups.
* Highlighted the highest-value retail transactions for business insights.
